# 演示指南

## 演示流程

### 1. 系统概览（2分钟）
- 介绍项目背景和解决的痛点
- 展示技术架构图
- 演示Docker容器化部署

### 2. 数据监控大屏（3分钟）
- 展示实时数据统计
- 演示品类分布图表
- 展示爆款指数趋势

### 3. 爆款分析（3分钟）
- 按品类筛选商品
- 展示爆款指数计算
- 分析热销商品特征

### 4. 差评分析（3分钟）
- 展示差评高频词
- 分析用户痛点
- 生成分析报告

### 5. 卖点推荐（2分钟）
- 基于差评分析
- 展示反向卖点提炼
- 生成优化建议

### 6. 相似爆款检索（2分钟）
- 输入关键词搜索
- 展示语义匹配结果
- 分析相似商品特征

### 7. AI文案生成（3分钟）
- 输入商品信息
- 选择文案风格
- 生成多种文案

### 8. 技术亮点（2分钟）
- 大数据技术栈整合
- AI + 大数据融合
- 双维度竞品分析

## 演示数据

使用Mock数据进行演示，包含：
- 100+商品数据
- 300+差评数据
- 覆盖8个品类

## 常见问题

**Q: 为什么使用Mock数据？**
A: 主要是为了演示系统功能，避免爬虫的法律风险和技术复杂度。

**Q: 系统性能如何？**
A: 在单机Docker环境下，可以处理10万+数据量，响应时间在1秒以内。

**Q: 如何扩展数据量？**
A: 修改mock-data脚本中的数量参数，重新生成数据即可。

## 演示准备

### 1. 启动服务
```bash
make build
make up
```

### 2. 生成测试数据
```bash
make mock-data
make init-data
```

### 3. 验证服务
```bash
# 验证后端
curl http://localhost:8000/health

# 验证前端
curl http://localhost:80

# 验证大数据组件
curl http://localhost:9200
curl http://localhost:16010
curl http://localhost:9870
```

### 4. 打开浏览器
- 访问 http://localhost 查看前端
- 访问 http://localhost:8000/docs 查看API文档

## 演示脚本

### 开场白
"大家好，我今天要演示的是抖音电商竞品智能分析与AI素材工厂。这个项目旨在为电商中小商家提供一站式竞品智能分析与AI广告素材生成服务，解决商家'不知道竞品什么卖得好、不知道文案怎么写、不知道产品卖点怎么优化'三大痛点。"

### 技术架构介绍
"这个项目采用了完整的大数据技术栈，包括Kafka用于数据缓冲、Spark用于分布式计算、HDFS用于分布式存储、HBase用于列式存储、Elasticsearch用于全文检索。前端使用Vue3和ECharts进行数据可视化，后端使用FastAPI提供API服务。"

### 功能演示
按照演示流程中的步骤逐一演示各功能模块。

### 创新点总结
"这个项目的创新点主要体现在以下几个方面：第一，双维度竞品分析，不仅分析爆款好在哪里，更分析竞品差在哪里；第二，反向卖点提炼，从竞品差评中挖掘用户痛点，转化为自身产品卖点；第三，完整大数据链路，实现了采集、缓冲、计算、存储、检索、应用、可视化的全链路打通；第四，AI与大数据融合，大数据提供洞察，AI提供生成能力，二者深度结合。"
