# 抖音电商竞品智能分析与AI素材工厂 - 项目设计文档

---

## 一、项目概述

### 1.1 项目定位

为电商中小商家提供**一站式竞品智能分析与AI广告素材生成服务**，解决商家"不知道竞品什么卖得好、不知道文案怎么写、不知道产品卖点怎么优化"三大痛点。

### 1.2 技术定位

基于大数据技术栈的完整项目，展示Kafka、Spark、HDFS、HBase、Elasticsearch等组件的整合应用能力。

### 1.3 开发约束

- **开发周期：** 2天（AI辅助编程）
- **部署方式：** Docker多容器伪分布式
- **项目类型：** 毕业设计级别
- **数据来源：** 官方API + Mock数据（主要用于演示）

## 三、数据模型

### 3.1 数据存储策略

| 存储组件 | 存储内容 | 原因 |
|---------|---------|------|
| **HBase** | 商品数据、差评数据、爆款特征 | 结构化数据，需要快速查询 |
| **Elasticsearch** | 商品全文索引、相似性检索 | 支持中文分词、语义匹配 |
| **HDFS** | 商品图片、视频素材 | 海量文件冷存储 |
| **Kafka** | 爬虫数据缓冲 | 解耦采集与处理 |

### 3.2 HBase表设计

**表1：商品表（product）**
```
RowKey: productId
列族: info
  - name: 商品名称
  - category: 品类
  - price: 价格
  - sales: 销量
  - rating: 好评率
  - hotScore: 爆款指数
  - imageUrl: 图片URL
  - createTime: 创建时间
```

**表2：差评表（review）**
```
RowKey: reviewId
列族: info
  - productId: 商品ID
  - content: 差评内容
  - rating: 评分
  - keywords: 关键词
  - sentiment: 情感倾向
  - createTime: 创建时间
```

**表3：文案表（copywriting）**
```
RowKey: copyId
列族: info
  - productId: 商品ID
  - type: 文案类型
  - content: 文案内容
  - tags: 标签
  - createTime: 创建时间
```

## 四、业务逻辑

### 4.1 核心业务流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    1. 数据采集流程                               │
│  抖音爬虫/Mock数据 → Kafka → Spark清洗 → HBase/ES存储          │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                    2. 爆款分析流程                               │
│  用户请求 → 查询HBase → 计算爆款指数 → 返回分析结果              │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                    3. 差评分析流程                               │
│  用户请求 → 查询差评数据 → Spark NLP分析 → 返回痛点报告         │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                    4. AI文案生成流程                             │
│  用户输入 → 查询相似爆款 → 调用大模型API → 生成文案              │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 功能模块详细流程

**模块1：数据监控大屏**
- 输入：无（自动展示）
- 处理：查询HBase/ES统计数据
- 输出：实时数据图表

**模块2：爆款分析**
- 输入：品类筛选条件
- 处理：查询商品数据，计算爆款指数
- 输出：爆款商品列表和分析报告

**模块3：差评分析**
- 输入：商品ID或品类
- 处理：查询差评数据，NLP提取关键词
- 输出：差评高频词、用户痛点报告

**模块4：卖点推荐**
- 输入：商品ID或品类
- 处理：分析竞品差评，反向提炼卖点
- 输出：产品优化建议列表

**模块5：相似爆款检索**
- 输入：用户描述（文本）
- 处理：ES语义检索
- 输出：相似爆款商品列表

**模块6：AI文案生成**
- 输入：商品信息+卖点+风格偏好
- 处理：调用大模型API
- 输出：多种风格的广告文案

## 五、接口设计

### 5.1 API接口规范

- **基础路径：** `/api/v1`
- **数据格式：** JSON
- **认证方式：** 无（课程作业简化）

### 5.2 核心接口列表

**1. 监控大屏接口**
```
GET /api/v1/dashboard/stats
返回：爬取量、品类分布、爆款统计
```

**2. 爆款分析接口**
```
GET /api/v1/products/hot
参数：category, limit, page
返回：爆款商品列表、爆款指数
```

**3. 差评分析接口**
```
GET /api/v1/reviews/analysis
参数：productId 或 category
返回：差评高频词、痛点报告
```

**4. 卖点推荐接口**
```
GET /api/v1/products/selling-points
参数：productId 或 category
返回：卖点建议列表
```

**5. 相似爆款检索接口**
```
GET /api/v1/products/search
参数：query（用户描述）
返回：相似爆款商品列表
```

**6. AI文案生成接口**
```
POST /api/v1/copywriting/generate
参数：productId, style, requirements
返回：生成的文案列表
```

### 5.3 接口响应格式

**成功响应：**
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

**错误响应：**
```json
{
  "code": 400,
  "message": "参数错误",
  "data": null
}
```

## 六、部署设计

### 6.1 Docker容器规划

| 服务 | 镜像 | 端口 | 说明 |
|------|------|------|------|
| **frontend** | nginx:alpine | 80 | Vue3前端 |
| **backend** | python:3.11 | 8000 | FastAPI后端 |
| **kafka** | confluentinc/cp-kafka | 9092 | 消息队列（KRaft模式） |
| **spark-master** | bitnami/spark | 7077 | Spark主节点 |
| **spark-worker** | bitnami/spark | 8081 | Spark工作节点 |
| **hdfs-namenode** | bde/hadoop-namenode | 9870 | HDFS主节点 |
| **hdfs-datanode** | bde/hadoop-datanode | 9864 | HDFS数据节点 |
| **hbase-master** | bde/hbase-master | 16010 | HBase主节点 |
| **hbase-regionserver** | bde/hbase-regionserver | 16020 | HBase区域服务器 |
| **elasticsearch** | elasticsearch:8.x | 9200 | 搜索引擎 |

### 6.2 启动顺序

```
1. 基础服务：Kafka（KRaft模式）
2. 计算服务：Spark Master → Spark Worker
3. 存储服务：HDFS NameNode → HDFS DataNode → HBase
4. 搜索服务：Elasticsearch
5. 应用服务：Backend → Frontend
```

## 十、创新点与亮点

1. **双维度竞品分析** - 不仅分析爆款好在哪里，更分析竞品差在哪里
2. **反向卖点提炼** - 从竞品差评中挖掘用户痛点，转化为自身产品卖点
3. **完整大数据链路** - 采集→缓冲→计算→存储→检索→应用→可视化全链路打通
4. **AI + 大数据融合** - 大数据提供洞察，AI提供生成能力，二者深度结合
5. **真实商业价值** - 场景贴近实际，中小商家真能用得上
