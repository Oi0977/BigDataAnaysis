# 电商数据洞察平台 - 数据流转说明

> 本文档完整介绍数据在项目中的流转过程，从 Mock 数据生成到 Spark 计算，再到最终存储和前端展示。

---

## 一、整体架构

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  Mock 数据   │───▶│    HDFS      │───▶│    Spark     │
│  生成脚本    │    │  原始存储    │    │  清洗 & 计算  │
└──────────────┘    └──────────────┘    └──────────────┘
                                               │
                                               ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   前端展示   │◀───│  FastAPI     │◀───│    HBase     │
│  Vue3+ECharts│    │  后端接口    │    │  结果存储    │
└──────────────┘    └──────────────┘    └──────────────┘
```

**技术组件：**
- **调度器**：Apache Airflow（每天凌晨 2:00 自动执行）
- **消息队列**：Kafka（数据传输通道）
- **分布式存储**：HDFS（原始数据 + 清洗数据）
- **计算引擎**：Spark（数据清洗 + 分析计算）
- **列式数据库**：HBase（分析结果存储）
- **搜索引擎**：Elasticsearch（全文检索）
- **后端**：FastAPI（Python）
- **前端**：Vue3 + ECharts

## 三、阶段 1：Mock 数据生成

由 Airflow 在宿主机上执行 Python 脚本，生成模拟的电商数据。

### 1.1 商品数据 (`products.json`)

**生成脚本：** `mock-data/generate_products.py`
**数据量：** 2,000 条（8 个品类，每品类 250 个商品）

**样例数据：**

```json
{
  "product_id": "P000001",
  "name": "小米 iPhone 15",
  "category": "手机",
  "brand": "小米",
  "price": 4582.43,
  "original_price": 5444.89,
  "image_url": "",
  "description": "小米手机，高端旗舰实力派，续航表现惊艳！",
  "shop_name": "小米手机旗舰店",
  "create_time": "2025-10-06T16:13:13"
}
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `product_id` | string | 商品唯一标识，格式 `P000001` |
| `name` | string | 商品名称（品牌 + 型号 + 后缀） |
| `category` | string | 品类：手机/电脑/服装/美妆/食品/家居/数码/运动 |
| `brand` | string | 品牌名称 |
| `price` | float | 当前售价（元），比 original_price 低 10%-50% |
| `original_price` | float | 原价（元） |
| `image_url` | string | 商品主图 URL（占位地址） |
| `description` | string | 商品卖点描述（NLP 模板生成） |
| `shop_name` | string | 店铺名称 |
| `create_time` | string | 商品上架时间（ISO 格式） |

**评分分布：** 好评(4-5星) 60% + 中评(3星) 20% + 差评(1-2星) 20%

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `review_id` | string | 评价唯一标识，格式 `R000001_001`（商品ID_序号） |
| `product_id` | string | 关联商品 ID |
| `content` | string | 评价内容（NLP 模板 + 同义词替换生成） |
| `rating` | int | 评分（1-5 星） |

### 1.3 日销量数据 (`monthly_sales.json`)

**生成脚本：** `mock-data/generate_monthly_sales.py`
**数据量：** 730,000 条（2,000 商品 × 365 天）

**生成策略：**
- 基础日销量：10-500 件/天
- 星期效应：周末比工作日高 20-50%
- 月度趋势：部分商品上升（5-15%/月），部分下降（3-10%/月）
- 大促活动：双 11（4-6 倍）、618（3-5 倍）、年货节（2.5-4 倍）

**样例数据：**

```json
{
  "product_id": "P000001",
  "date": "2025-06-01",
  "daily_sales": 398,
  "daily_amount": 1823807.14
}
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `product_id` | string | 关联商品 ID |
| `date` | string | 日期（YYYY-MM-DD） |
| `daily_sales` | int | 当日销量（件） |
| `daily_amount` | float | 当日销售额（元）= daily_sales × price |

## 四、阶段 2：初始化存储（HBase + Elasticsearch）

Mock 数据生成后，由 Airflow 执行 `scripts/init_data.py`，将原始数据**同时写入 HBase 和 Elasticsearch**。

### 4.1 HBase 写入

将三种原始数据批量写入 HBase 对应表：

| 数据 | HBase 表 | RowKey | Column Family |
|------|---------|--------|--------------|
| products.json | `product` | product_id | `info`（name, category, brand, price 等） |
| reviews.json | `review` | review_id | `info`（productId, content, rating 等） |
| monthly_sales.json | `monthly_sales` | product_id_date | `data`（dailySales, dailyAmount） |

### 4.2 Elasticsearch 写入

将商品基础信息索引到 ES，用于**全文搜索**和**相似商品推荐**。

**ES 索引结构：**

```
索引名: products
├── productId     (keyword)      商品ID
├── name          (text, ik_max_word)  商品名称（中文分词）
├── category      (keyword)      品类
├── brand         (keyword)      品牌
├── description   (text, ik_max_word)  商品描述（中文分词）
├── price         (float)        售价
├── originalPrice (float)        原价
└── shopName      (keyword)      店铺名称
```

**ES 在项目中的用途：**

| 功能 | API 接口 | 说明 |
|------|---------|------|
| 全文搜索 | `GET /api/products/search?query=手机` | 按 name、description、category、brand 加权匹配 |
| 相似推荐 | `GET /api/products/{id}/similar` | 根据商品名称和描述查找相似商品 |

## 五、阶段 3：HDFS 存储 + Spark 数据清洗

### 5.1 上传到 HDFS

Airflow 将生成的 JSON 文件上传到 HDFS 的原始目录：

```
/douyin/raw/
├── products/products.json         2,000 条
├── reviews/reviews.json          22,970 条
└── monthly_sales/monthly_sales.json  730,000 条
```

### 4.2 Spark 数据清洗

**作业名称：** `EcommerceDataCleaning`
**脚本：** `spark-jobs/data_cleaning.py`
**执行命令：**
```bash
spark-submit --master spark://spark-master:7077 data_cleaning.py
```

**清洗规则：**

| 数据类型 | 清洗规则 |
|---------|---------|
| 商品数据 | 去除首尾空格、价格负值设为 0、过滤空名称/空ID记录 |
| 评价数据 | 去除首尾空格、评分限制 1-5 范围、过滤空内容/空ID记录 |
| 日销量数据 | 销量/金额负值设为 0、过滤空日期/空ID记录 |

**清洗输出：**

```
/douyin/cleaned/
├── products/        清洗后商品数据
├── reviews/         清洗后评价数据
└── monthly_sales/   清洗后日销量数据
```

**数据流转：**
```
/douyin/raw/products/    ──Spark清洗──▶  /douyin/cleaned/products/
/douyin/raw/reviews/     ──Spark清洗──▶  /douyin/cleaned/reviews/
/douyin/raw/monthly_sales/ ──Spark清洗──▶ /douyin/cleaned/monthly_sales/
```

## 六、阶段 4：Spark 分析计算（三个并行作业）

清洗完成后，三个 Spark 作业**并行执行**。

### 6.1 商品分析作业 (`product_analysis_job.py`)

**作业名称：** `EcommerceProductAnalysis`
**输入：** HDFS `/douyin/cleaned/` 下的三种清洗数据
**输出：** HDFS `/douyin/processed/product_analysis` + HBase `product_analysis` 表

**计算逻辑：**

```
                    ┌─────────────┐
                    │ 商品基础数据 │
                    └──────┬──────┘
                           │
    ┌──────────────────────┼──────────────────────┐
    ▼                      ▼                      ▼
┌─────────┐         ┌───────────┐          ┌──────────┐
│ 日销量   │         │ 评价数据   │          │ 关键词   │
│ 聚合     │         │ 聚合      │          │ 提取     │
└────┬────┘         └─────┬─────┘          └────┬─────┘
     │                    │                      │
     ▼                    ▼                      │
┌──────────┐        ┌───────────┐                │
│ total_sales│      │review_count│               │
│ monthly_  │       │avg_rating  │               │
│ growth    │       │positive_rate│              │
└────┬────┘        │negative_rate│               │
     │             └─────┬─────┘                │
     │                   │                      │
     └─────────┬─────────┘                      │
               ▼                                │
     ┌──────────────────┐                       │
     │  品类内比例法     │                       │
     │  计算爆款指数     │                       │
     └────────┬─────────┘                       │
              │                                 │
              └──────────┬──────────────────────┘
                         ▼
                 ┌──────────────┐
                 │  最终结果     │
                 │  hot_score   │
                 │  top_tags    │
                 └──────────────┘
```

**爆款指数公式（品类内比例法）：**

```python
# 各维度得分（归一化到 0-1）
sales_score  = total_sales / 品类内最大total_sales
growth_score = monthly_growth / 100（截断到 0-1）
rating_score = avg_rating / 5.0
review_score = review_count / 品类内最大review_count

# 综合爆款指数
hot_score = 0.4 × sales_score + 0.3 × growth_score + 0.2 × rating_score + 0.1 × review_score
```

**权重设计理由：**
- 销量（40%）：最直接反映市场表现
- 增长率（30%）：反映商品上升趋势
- 评分（20%）：反映用户满意度
- 评价数（10%）：反映用户参与度

**所有计算字段（含中间量）：**

| 字段 | 类型 | 计算方式 | 写入 HBase？ | 说明 |
|------|------|---------|-------------|------|
| `product_id` | string | 原始 | RowKey | 商品唯一标识 |
| `total_sales` | int | SUM(daily_sales) | **否** | 近 12 月总销量（中间量） |
| `latest_month_sales` | int | 最近月 SUM | **否** | 最近一个月销量（中间量） |
| `last_month_sales` | int | 上月 SUM | **否** | 上个月销量（中间量） |
| `monthly_growth` | float | (最近月-上月)/上月×100 | **是** `metrics:monthlyGrowth` | 月增长率（%） |
| `review_count` | int | COUNT(*) | **是** `metrics:reviewCount` | 评价总数 |
| `avg_rating` | float | AVG(rating) | **是** `metrics:avgRating` | 平均评分（1-5） |
| `positive_rate` | float | COUNT(rating≥4)/总数 | **是** `metrics:positiveRate` | 好评率（0-1） |
| `negative_rate` | float | COUNT(rating≤2)/总数 | **是** `metrics:negativeRate` | 差评率（0-1） |
| `category_max_sales` | int | Window MAX | **否** | 品类内最大总销量（窗口函数） |
| `category_max_reviews` | int | Window MAX | **否** | 品类内最大评价数（窗口函数） |
| `sales_score` | float | total_sales / category_max_sales | **否** | 销量维度得分（0-1） |
| `growth_score` | float | monthly_growth / 100 截断 | **否** | 增长维度得分（0-1） |
| `rating_score` | float | avg_rating / 5.0 | **否** | 评分维度得分（0-1） |
| `review_score` | float | review_count / category_max_reviews | **否** | 评价维度得分（0-1） |
| `hot_score` | float | 加权求和 | **是** `metrics:hotScore` | 综合爆款指数（0-1） |
| `top_tags` | list | jieba 提取 | **是** `metrics:topTags` | 高频关键词 Top 10 |

**样例计算结果：**

```json
{
  "product_id": "P000001",
  "total_sales": 181367,
  "monthly_growth": 12.5,
  "review_count": 9,
  "avg_rating": 4.11,
  "positive_rate": 0.7778,
  "negative_rate": 0.1111,
  "hot_score": 0.4986,
  "top_tags": ["华为", "OPPO", "Find", "X7", "Pro", "Mate60", "卫星通信", "功能", "救急", "两次"]
}
```

> **注意：** `score_breakdown`（sales/growth/rating/review 四个维度得分）在 Spark 作业中计算了但**未写入 HBase**。只有 Python 备用脚本 `compute_analysis.py` 才会写入 `score_breakdown`。

### 6.2 评价 NLP 分析作业 (`nlp_analysis_job.py`)

**作业名称：** `EcommerceNLPAnalysis`
**输入：** HDFS `/douyin/cleaned/reviews`
**输出：** HDFS `/douyin/processed/review_analysis` + HBase `review_analysis` 表

**计算逻辑：**

```
┌─────────────────┐
│  清洗后评价数据  │
└────────┬────────┘
         │
    ┌────┴────┐
    ▼         ▼
┌────────┐ ┌──────────┐
│ jieba  │ │SnowNLP   │
│ 分词   │ │情感分析   │
└───┬────┘ └────┬─────┘
    │           │
    ▼           ▼
┌────────────────────┐
│  按商品聚合        │
│  - 高频关键词      │
│  - 情感分布        │
│  - 评分分布        │
└────────────────────┘
```

**NLP 处理流程：**

1. **jieba 关键词提取**：对每条评价分词，过滤停用词和单字词，提取关键词
2. **SnowNLP 情感分析**：计算情感得分（0-1），`< 0.3` 为 negative，`0.3-0.7` 为 neutral，`> 0.7` 为 positive
3. **按商品聚合**：合并每个商品的所有评价，统计词频取 Top 10

**所有计算字段：**

| 字段 | 类型 | 计算方式 | 写入 HBase？ | 说明 |
|------|------|---------|-------------|------|
| `product_id` | string | 原始 | RowKey | 商品唯一标识 |
| `review_count` | int | COUNT(*) | **否**（JSON 里有） | 评价总数 |
| `avg_rating` | float | AVG(rating) | **否**（JSON 里有） | 平均评分 |
| `high_freq_keywords` | list | jieba 词频统计 | **是** `stats:highFreqKeywords` | 关键词+词频 JSON |
| `sentiment_distribution` | dict | SnowNLP 分类统计 | **是** `stats:sentimentDistribution` | 情感分布 JSON |
| `rating_distribution` | dict | 评分分组统计 | **是** `stats:ratingDistribution` | 评分分布 JSON |

**样例结果：**

```json
{
  "product_id": "P000001",
  "review_count": 9,
  "avg_rating": 4.11,
  "high_freq_keywords": [
    {"keyword": "X100", "count": 3},
    {"keyword": "物流", "count": 2},
    {"keyword": "夜景", "count": 2}
  ],
  "sentiment_distribution": {
    "positive": 7,
    "neutral": 1,
    "negative": 1
  },
  "rating_distribution": {
    "1": 0, "2": 1, "3": 1, "4": 2, "5": 5
  }
}
```

**HBase 表结构：**

```
review_analysis 表:
  RowKey: product_id
  Column Family: stats
    - stats:highFreqKeywords      高频关键词（JSON）
    - stats:sentimentDistribution 情感分布（JSON）
    - stats:ratingDistribution    评分分布（JSON）
    - stats:updateTime           更新时间
```

### 6.3 趋势聚合作业 (`trend_aggregation_job.py`)

**作业名称：** `EcommerceTrendAggregation`
**输入：** HDFS `/douyin/cleaned/monthly_sales` + `/douyin/cleaned/products`
**输出：** HDFS `/douyin/processed/trend_data` + HBase `trend_data` 表

**计算逻辑：**

```
┌──────────────────┐
│ 日销量数据        │
│ (730,000 条)     │
└────────┬─────────┘
         │
         ▼  按 product_id + month 聚合
┌──────────────────┐
│ 月度销量          │
│ (2000×12=24,000) │
└────────┬─────────┘
         │
    ┌────┴────┐
    ▼         ▼
┌────────┐ ┌────────────┐
│ 商品   │ │ 品类       │
│ 趋势   │ │ 趋势聚合   │
└───┬────┘ └─────┬──────┘
    │            │
    ▼            ▼
┌────────┐ ┌────────────┐
│ sales_ │ │ category_  │
│ trend  │ │ trend      │
└────────┘ └────────────┘
```

**所有计算字段：**

| 字段 | 类型 | 计算方式 | 写入 HBase？ | 说明 |
|------|------|---------|-------------|------|
| `product_id` | string | 原始 | RowKey | 商品唯一标识 |
| `sales_trend` | list | 日→月聚合 | **否**（JSON 里有） | 近 12 月月度销量数组 |
| `amount_trend` | list | 日→月聚合 | **否** | 近 12 月月度销售额数组 |
| `monthly_detail` | list | 日→月聚合 | **否** | 每月详情 [{month, sales, amount}] |
| `total_sales` | int | SUM(月度) | **是** `monthly:totalSales` | 总销量 |
| `total_amount` | float | SUM(月度) | **是** `monthly:totalAmount` | 总销售额 |
| `avg_monthly_sales` | float | total/月数 | **是** `monthly:avgMonthlySales` | 月均销量 |
| `category_trend` | dict | 按品类聚合 | **是**（品类维度） | 各品类月度销量趋势 |

**聚合步骤：**
1. 从 `date` 字段提取月份（`YYYY-MM`）
2. 按 `product_id + month` 聚合日销量为月度销量
3. 为每个商品生成近 12 个月的销量数组
4. 按品类聚合所有商品的月度销量

**HBase 表结构：**

```
trend_data 表:
  RowKey: product_id
  Column Family: monthly
    - monthly:totalSales        总销量
```

## 十二、关键技术点

### 12.1 品类内比例法

爆款指数采用**品类内比例法**计算，确保各品类之间具有可比性：

```python
# 同一品类内，销量最高的商品 sales_score = 1.0
# 其他商品按比例获得 0-1 之间的值
sales_score = total_sales / category_max_sales

# 评价数同理
review_score = review_count / category_max_reviews
```

**效果：**
- 手机品类中销量最高的商品：`sales_score = 1.0`
- 手机品类中销量第二的商品：`sales_score ≈ 0.83`
- 服装品类中销量最高的商品：`sales_score = 1.0`（独立计算）

### 12.2 爆款定义

前端展示的"爆款"定义为**每个品类 Top 20%** 的商品：

```python
# 每个品类按 hot_score 降序排列，取前 20%
top_count = max(1, int(len(category_products) * 0.2))
```

```python
    # 连接断开时自动重连并重试
```
