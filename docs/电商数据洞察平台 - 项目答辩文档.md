## 一、项目概述

### 1.1 项目名称

**电商数据洞察平台**

### 1.2 项目定位

面向电商行业的**一站式数据分析与洞察平台**，通过大数据技术栈帮助商家解决以下核心问题：

| 痛点问题 | 解决方案 | 技术支撑 |
|----------|----------|----------|
| 竞品分析难，不知道什么卖得好 | 爆款指数计算 + 多维度排行榜 | Spark 分布式计算 |
| 用户需求洞察难，不知道用户在意什么 | NLP 评价分析 + 情感分析 | jieba + SnowNLP |
| 趋势把握难，不知道市场走向 | 月度销量趋势聚合 + 可视化 | Spark + ECharts |
| 数据分散，难以统一管理 | 统一数据平台 + 全文检索 | HBase + Elasticsearch |

### 1.3 数据规模

| 数据类型 | 数量 | 说明 |
|----------|------|------|
| 商品数据 | 100+ 条 | 覆盖 8 个品类 |
| 评价数据 | 1000+ 条 | 包含好评、中评、差评 |
| 日销量数据 | 36000+ 条 | 近 12 个月每日销量 |
| 品类覆盖 | 8 个 | 手机、电脑、服装、美妆、食品、家居、数码、运动 |

### 1.4 项目价值

**技术价值：**
- 完整的大数据技术栈实践（8+ 大数据组件）
- AI 与大数据的深度融合
- 企业级自动化调度方案

**应用价值：**
- 帮助商家了解市场竞品表现
- 从评价中挖掘用户真实需求
- 基于数据驱动的产品优化建议

### 3.3 功能2：爆款分析

**作用**：分析市场上的爆款商品，计算爆款指数

**核心算法 - 爆款指数（hot_score）**：

```
hot_score = (0.4 × 销量分 + 0.3 × 增长分 + 0.2 × 评分分 + 0.1 × 评价分) × 1000
```

| 维度 | 权重 | 计算方式 | 业务含义 |
|------|------|----------|----------|
| 销量分 | 40% | 分段归一化（<1000→0.2, <5000→0.4, ...） | 反映销售能力 |
| 增长分 | 30% | monthly_growth / 100 | 反映增长趋势 |
| 评分分 | 20% | avg_rating / 5.0 | 反映用户满意度 |
| 评价分 | 10% | 分段归一化（<10→0.2, <50→0.4, ...） | 反映市场热度 |

**可视化内容**：
- 爆款排行榜（柱状图）
- 销量 vs 爆款指数散点图（按品类分色）
- 品类 Top 商品占比（饼图）
- 月度销量趋势（折线图）

### 3.4 功能3：差评分析

**作用**：从差评中挖掘用户痛点，生成分析报告

**分析流程**：
```
评价数据 (1000+条)
    │
    ├─→ jieba 分词 → 停用词过滤 → 关键词提取 (Top 10)
    │
    ├─→ SnowNLP 情感分析 → 情感分类 (positive/neutral/negative)
    │
    ├─→ 评分分布统计 (1-5分)
    │
    └─→ 按商品聚合 → 生成分析报告
```

**可视化内容**：
- 高频关键词排行（带进度条）
- 用户痛点 TOP5
- 情感分布饼图（正面/中性/负面）
- 评分分布柱状图

### 3.5 功能4：相似检索

**作用**：基于 Elasticsearch 实现商品相似检索

**技术实现**：
```json
{
  "query": {
    "multi_match": {
      "query": "手机",
      "fields": ["name^3", "description^2", "category", "brand"]
    }
  }
}
```

**权重设计**：
- `name^3`：商品名称权重最高
- `description^2`：描述次之
- `category`、`brand`：分类和品牌作为辅助匹配

**用途**：帮助商家快速找到竞品进行对比分析

### 4.1 Spark 核心算法：爆款指数计算

**文件**：`spark-jobs/product_analysis_job.py`

**为什么重要**：这是整个平台的核心算法，体现了 Spark 分布式计算能力

```python
def compute_hot_score(spark, products_df, sales_metrics, review_metrics):
    """
    计算爆款指数 hot_score

    公式:
      hot_score = (0.4*sales_score + 0.3*growth_score +
                   0.2*rating_score + 0.1*review_score) * 1000
    """
    # 1. 关联商品、销量、评价数据
    combined = products_df \
        .join(sales_metrics, "product_id", "left") \
        .join(review_metrics, "product_id", "left")

    # 2. 分段归一化计算各维度分数
    combined = combined \
        .withColumn("sales_score",
                    when(col("total_sales") < 1000, lit(0.2))
                    .when(col("total_sales") < 5000, lit(0.4))
                    .when(col("total_sales") < 10000, lit(0.6))
                    .when(col("total_sales") < 50000, lit(0.8))
                    .otherwise(lit(1.0))) \
        .withColumn("growth_score",
                    when(col("monthly_growth") / 100 > 1.0, lit(1.0))
                    .when(col("monthly_growth") / 100 < 0.0, lit(0.0))
                    .otherwise(col("monthly_growth") / 100)) \
        .withColumn("rating_score", col("avg_rating") / 5.0) \
        .withColumn("review_score",
                    when(col("review_count") < 10, lit(0.2))
                    .when(col("review_count") < 50, lit(0.4))
                    .when(col("review_count") < 100, lit(0.6))
                    .when(col("review_count") < 200, lit(0.8))
                    .otherwise(lit(1.0)))

    # 3. 加权计算最终爆款指数
    combined = combined.withColumn(
        "hot_score",
        col("sales_score") * 0.4 +
        col("growth_score") * 0.3 +
        col("rating_score") * 0.2 +
        col("review_score") * 0.1
    )
    return combined
```

**答辩要点**：
- 使用 PySpark 的 `when/otherwise` 实现分段归一化
- 权重设计体现业务逻辑（销量最重要，评价量次要）
- 在分布式环境下批量计算 100+ 商品的爆款指数

### 4.3 趋势聚合：日销量转月度趋势

**文件**：`spark-jobs/trend_aggregation_job.py`

**为什么重要**：展示时序数据处理能力

```python
def generate_sales_trend(spark, monthly_agg_df):
    """生成近12个月的月度销量趋势数组"""
    # 获取所有月份（排序后取最近12个）
    all_months = sorted([
        row["month"] for row in monthly_agg_df.select("month").distinct().collect()
    ])
    recent_12_months = all_months[-12:] if len(all_months) > 12 else all_months

    # 按商品聚合月度数据
    product_monthly = monthly_agg_df.groupBy("product_id").agg(
        collect_list(struct("month", "monthly_sales")).alias("monthly_data")
    ).collect()

    results = []
    for row in product_monthly:
        month_map = {item["month"]: item["monthly_sales"] for item in row["monthly_data"]}

        # 生成近12个月的销量数组（缺失月份填充0）
        sales_trend = [int(month_map.get(month, 0)) for month in recent_12_months]

        results.append({
            "product_id": row["product_id"],
            "sales_trend": sales_trend,  # [1200, 1350, 1500, ...]
            "total_sales": sum(sales_trend),
        })
    return results, recent_12_months
```

**答辩要点**：
- 使用 `collect_list(struct(...))` 组合复杂数据结构
- 时间序列补齐（缺失月份填充0）
- 为前端 ECharts 提供标准化的时序数据格式

### 4.4 Airflow DAG：自动化调度编排

**文件**：`dags/ecommerce_pipeline.py`

**为什么重要**：展示企业级调度能力

```python
# DAG 定义：每天凌晨2点执行
with DAG(
    'ecommerce_etl_pipeline',
    schedule_interval='0 2 * * *',  # Cron表达式
    retries=3,                       # 失败重试3次
    retry_delay=timedelta(minutes=5),
    execution_timeout=timedelta(hours=2),
) as dag:

    # 阶段1: 并行生成数据
    start >> [generate_products, generate_reviews, generate_sales]

    # 阶段2: 初始化存储
    [generate_products, generate_reviews, generate_sales] >> init_hbase_es

    # 阶段3: 上传HDFS
    init_hbase_es >> create_hdfs_dirs >> [upload_products, upload_reviews, upload_sales]

    # 阶段4: Spark清洗
    [upload_products, upload_reviews, upload_sales] >> spark_cleaning

    # 阶段5: Spark分析（3个作业并行）
    spark_cleaning >> [spark_product_analysis, spark_nlp_analysis, spark_trend_aggregation]

    # 阶段6: 计算指标
    [spark_product_analysis, spark_nlp_analysis, spark_trend_aggregation] >> compute_analysis
```

### 4.5 HBase 服务：自动重连装饰器

**文件**：`backend/app/core/hbase_service.py`

**为什么重要**：展示生产级的防御性编程

```python
def _retry_on_disconnect(func):
    """装饰器：HBase连接断开时自动重连并重试，最多重试2次"""
    @functools.wraps(func)
    def wrapper(self, *args, **kwargs):
        for attempt in range(3):
            try:
                return func(self, *args, **kwargs)
            except Exception as e:
                error_msg = str(e).lower()
                is_conn_err = any(kw in error_msg for kw in [
                    'timed out', 'connection', 'broken', 'socket', 'reset'
                ])
                if is_conn_err and attempt < 2:
                    print(f"[HBase重试] {func.__name__} 第{attempt+1}次失败，重连中")
                    self.connect()  # 自动重连
                else:
                    raise
    return wrapper

# 所有 HBase 操作都使用该装饰器
@_retry_on_disconnect
def get_all_products(self):
    ...

@_retry_on_disconnect
def write_product_analysis(self, data):
    ...
```

**答辩要点**：
- 装饰器模式统一处理连接异常
- 自动识别网络错误关键词
- 最多重试2次，避免无限循环
- 体现对分布式系统不稳定性的防御性编程

### 4.6 ES 搜索：多字段权重检索

```python
def search_products(self, query: str, limit: int = 10):
    search_query = {
        "query": {
            "multi_match": {
                "query": query,
                "fields": ["name^3", "description^2", "category", "brand"]
            }
        },
        "size": limit
    }
    response = self.client.search(index=self.index_name, body=search_query)
    return [hit['_source'] for hit in response['hits']['hits']]
```

**答辩要点**：

- 字段权重设计（name^3 > description^2）
- 返回相关性评分，便于排序
